# 计算机网络作业自动生成工具 (Gemini / OpenAI 版)

本工具是一个基于大语言模型 (LLM) 的自动化作业辅助系统，专门用于解析计算机网络作业 PDF，通过深度推理 (CoT) 、每题搜索和多轮审阅机制生成高质量答案，并自动填充至 Word 模板。

## 核心功能

- **智能 PDF 结构化解析**：利用 AI 将乱序的 PDF 文本自动分类为选择题、简答题和编程题。
- **选择题容量扩展**：第一部分选择题按模板支持最多 30 题（超出部分不会写入模板）。
- **题目截图自动生成**：脚本会把 PDF 中每道题自动截图并保存到 `problems/` 目录。**支持跨页题目自动识别与垂直合并**，并利用 `numpy` 智能跳过页眉页脚，确保题目截图的完整性。后续答题与审阅阶段会附上对应题图，显著提升复杂题目的理解准确度。
- **参考答案最终全量修复**：可通过命令行参数传入一个参考答案 PDF（格式与报告一致），脚本会按题号自动截图。在所有题目初稿完成后，系统会进行一次“最终全量审阅修复”（complex 模型），对照参考截图统一修正。最终修复阶段强调基于原文最小改动、减少不必要括号解释，并尽量避免 AI 套话文风。
- **Chain-of-Thought (CoT) 推理**：模型在给出答案前，必须在 `<thought>` 标签内明确列出【考察知识点】并展示逻辑推导过程。
- **循环审阅与事实核查机制**：
  - **每题搜索**：在解答每一道选择题和简答题前，AI 会先调用 **SearxNG** 进行背景搜索。
  - **全员工具接入**：答题 AI 与审阅 AI 均接入了 **Python Interpreter** 和 **SearxNG Search**。
  - **自动迭代优化**：若审阅未通过（如文风不正式、逻辑错误等），系统会自动反馈具体原因让答题 AI 修正（最高 10 轮）。
- **去 AI 痕迹化表达**：
  - **严谨实验报告文风**：语气正式、专业且客观，符合学术规范。
  - **语言纯净**：全中文回答（除必要术语），严禁出现 Markdown 语法、大量括号解释或中英对照。
- **交互式确认**：在最终内容生成后保留人工反馈环节，可继续微调最终作业。

## 快速开始

### 1. 环境准备
确保已安装 [uv](https://github.com/astral-sh/uv)。

```bash
uv sync
```

### 2. SearxNG 配置 (关键)
为了使搜索功能正常工作，必须开启本地 SearxNG 的 JSON API 支持：
1. 编辑 SearxNG 的 `settings.yml`。
2. 确保 `search` 部分下的 `formats` 包含 `json`：
   ```yaml
   search:
       formats:
           - html
           - json
   ```
3. 重启 SearxNG 服务。

### 3. 配置工具
1. 复制模板：`copy config-template.toml config.toml`
2. 编辑 `config.toml`：
   - `llm.simple`: 用于解析和审阅的轻量级模型。
   - `llm.complex`: 用于核心推理和答题的高性能模型。
   - `searxng`: 设置 base_url（例如 `http://localhost:8089/`）。
   - `student_info`: 填写你的基本信息。

## 使用方法

### 1. 放置文件
- 将作业 PDF 放入项目根目录。
- 确保根目录下有 `template.docx` (Word 模板) 和 `project_prompt.txt` (编程题系统提示词)。

### 2. 启动脚本
```bash
uv run python main.py [作业文件名.pdf]
```

可选：传入参考答案 PDF（仅用于最后一次全量审阅修复）

```bash
uv run python main.py [作业文件名.pdf] --reference-pdf [参考答案文件名.pdf]
```

示例：

```bash
uv run python main.py 第6课作业.pdf --reference-pdf 第6课参考答案.pdf
```

### 3. 交互流程
1. **自动截图**：启动后会先解析作业 PDF，并在 `problems/` 下生成每题截图。
2. **可选参考对照**：若传入 `--reference-pdf`，会按相同题号生成参考答案截图。
3. **监控自动答题**：AI 会逐题进行搜索、CoT 推理和内部审阅，并附带题目截图作为上下文。
4. **最终全量审阅修复**：所有题目初稿完成后，系统会调用 complex 模型对照参考截图进行一次统一修复。
5. **输入编程题链接**：脚本会针对每道编程题弹出提示，请根据 AI 生成的“项目建议提示词”在 Gitee 完成项目后，将链接输入控制台。
6. **最终反馈与生成**：所有题目完成后，你可以最后预览一次 context 内容。若不满意可以输入修改意见进行全局调整，输入 `OK` 后将生成最终的 `{作业名称}.docx`。

## 注意事项
- 系统内置了 `_clean_markdown` 兜底机制，会自动剔除答案中残留的 `**加粗**` 等符号。
- 网络连接不稳定时，系统会自动进行重试（当前实现上限为 1000 次，随重试次数逐步增加等待时间）。
- 生成结果保存到 Word 时，如目标文件正被占用（例如在 Word 中打开），程序会自动重试；若仍失败，会自动切换为带时间戳的新文件名后再保存。
- 严禁在 Python 解释器中使用文件读写或访问互联网。
